<b>В Java есть два вида сообщений о ошибках:</b>
<b>1.Error</b>
критические ошибки, которые обычно не возможно исправить, и которые приводят к завершению программы, такие как OutOfMemoryError. Их можно перехватить с помощью catch, но в этом редко бывает смысл.
<b>2.Exception </b>
Ошибки приложения, которые можно обработать. 
<b>Они делятся на:</b>
<b>checked</b>
Exception, которые обязательно нужно либо перехватить в методе с помощью catch или переслать дальше с помощью throws (указать данный Exception или его суперкласс в секции throws метода)
<b>unchecked</b>
Exception, которые можно не перехватывать и не указывать в секции throws метода,

<b>Структура наследования:</b>
<b>1.Throwable</b>
Любые ошибки и exception
<b>1.1 Error</b>
Критические ошибки, которые обычно не обрабатываются и завершают программу,
<b>1.2 Exception</b>
Все exception, по умолчанию все наследуемые классы checked exception (кроме тех кто наследуются от RuntimeException)
<b>1.2.1 RuntimeException </b>
Все наследуемые от этого классы — unchecked exception.
